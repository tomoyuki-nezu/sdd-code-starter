## 背景・目的

新規プロジェクトの初期化に必要な決定事項を整理します。回答内容に基づいてプロジェクト構造・設定ファイル・CI/CD を自動生成します。

## 前提（固定ではない）

- 開発の Single Source of Truth は **仕様書（`spec/`）** とし、SDD（仕様書ドリブン開発）で進めることを推奨する
- CI/CD のホスト、ローカル開発の起動方法、ドキュメント方針は **下記 Q5〜Q7 で決める**。このスターターは GitHub Actions や `docker-compose.yml` に例示しやすいが、回答と異なる場合は生成物を回答に合わせる

### Q2. 技術スタック

**質問内容**
使用する技術スタックを教えてください。

**選択肢（言語）**

- B) TypeScript / Node.js
- C) その他（自由記述）

**選択肢（実行環境）**

**質問内容**

- C) その他（自由記述）

- クラウドプロバイダー: AWS / GCP / Azure / その他 / なし

### Q5. CI/CD の要件

**質問内容**
CI/CD の要件を教えてください。

**回答形式**

- テストの要件: ユニットテストのみ / ユニット + E2E / その他
