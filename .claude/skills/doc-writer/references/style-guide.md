# Document Writer — スタイルガイド詳細

SKILL の本文は `SKILL.md`。このファイルは必要時に読み込む参照層。

## 言語

- ドキュメントは日本語で記述すること
- コミットメッセージは英語（git-workflow Skill に従う）

## 機密情報の取り扱い

実値は書かず、チームで決めたプレースホルダに置換する（例）：

- クラウドアカウント ID → `YOUR_ACCOUNT_ID` 形式
- 認証用ロール/サービスアカウント → `YOUR_ROLE` 形式
- SSO / 認証 URL → `YOUR_AUTH_URL` 形式
- API エンドポイント URL → `YOUR_API_URL` 形式
- プロファイル名 → `YOUR_PROFILE_NAME` 形式

## ドキュメントの種類と配置

| 種類 | 配置先 | 説明 |
|---|---|---|
| 汎用ドキュメント | docs/universal/ | どのプロジェクトでも使える |
| プロジェクト固有 | docs/project/ | このプロジェクト専用 |
| README | ルート | プロジェクトの入口 |

## README.md の標準セクション

1. プロジェクト概要
2. システム構成図
3. 前提条件
4. 初回セットアップ手順
5. ローカル開発のワークフロー
6. Claude Code を使った開発フロー
7. デプロイ後の確認方法
8. トラブルシューティング
9. インフラリソース一覧

## 記述スタイル

- 箇条書きとテーブルを活用する
- コマンド例は必ずコードブロックで記載
- トラブルシューティングは「症状→原因→解決」の形式
- システム構成図はテキスト図（ASCII art）で表現
