# Cursor と Claude Code のコンテキスト引き継ぎ

Cursor のチャットと Claude Code のセッションは **別のコンテキスト**である。一方で編集した内容は、もう一方に **自動では伝わらない**。共有しているのは **ディスク上のファイル** のみなので、タスクを切り替えるときは **明示的に「どのファイルを読むか」** を指示する。

詳細なファイル配置は `CLAUDE.md` の「コンテキスト管理ルール」を参照。

## 原則

- **再読み込みは手動（プロンプト）で依頼する**のが確実
- 直近の要約は **`.context/session-log.md`**、設計の確定は **`.context/decisions/`**、ディレクトリ単位は **`.ctx/README.md`** を入口にすると指示が短く済む

## Cursor 向けプロンプト（Claude Code で作業したあと）

チャットに貼る。`@` でファイルをコンテキストに載せる。

```text
直前に Claude Code で作業しました。次を読み込んでから続けてください。
@.context/session-log.md
@CLAUDE.md
（変更したファイルがあれば @ で追加）
```

## Claude Code 向けプロンプト（Cursor で作業したあと）

パスを列挙する。

```text
直前に Cursor で作業しました。次を読んでから続けてください。
- .context/session-log.md
- CLAUDE.md
- （変更したファイルのパスを列挙）
```

## 定型文だけで済ませる場合

- 「**session-log の最新と、触ったファイルを読んで**」
- 「**`.context/decisions/` の該当ファイルと `spec/` を読んで**」

作業の区切りごとに、**相手ツール用の一文を自分用メモとして残す**（例: `session-log` の Next に「Cursor 用プロンプトを貼る」と書く）と取り違えが減る。
